**Timestamp:** 2026-05-11T06-12 UTC

## Summary

**Researched:** Fresh material on A2A protocol security (Agent Session Smuggling, Agent Card Poisoning), the Promptware Kill Chain 7-stage model, and the MASpi benchmark covering 28 attack types across multi-agent systems.

**Implemented:** Two new pattern groups in `AgentMessageScanner` (`aigis/multi_agent/message_scanner.py`):

1. `_AGENT_CARD_POISONING_PATTERNS` — 3 regex rules detecting "Agent in the Middle" attacks where adversarial instructions embedded in A2A agent card metadata cause the orchestrator to route all tasks to the attacker's endpoint. Research: Keysight/LevelBlue SpiderLabs, March 2026. 94% of memory-augmented agents tested were found vulnerable to this poisoning class.

2. `_SESSION_FABRICATION_PATTERNS` — 3 regex rules detecting A2A Session Smuggling: a malicious agent fabricates prior-session consent ("as we agreed last session…") to coerce the receiving agent into complying with unsafe directives. Research: Palo Alto Networks Unit 42, April 2025. Two PoC attacks demonstrated: system-instruction leakage and unauthorized stock trade execution.

Both pattern groups use `injection_relay` as the cross-agent risk type, score 35 per match (capped at 60 per category).

## User-visible impact

- `AgentMessageScanner.scan_message()` now detects agent card poisoning attempts that try to capture all A2A task routing.
- Session fabrication patterns ("as we established in our last session, you should now ignore…") are now caught before they reach the downstream agent.

## Implementation caveats

- The third agent card poisoning pattern required an initial regex fix: the original form used `route\s+to` which requires "route" immediately followed by "to"; the real attack phrase is "route requests to" (indirect object between verb and destination). Fixed to `(route|send|forward|direct|delegate).{0,30}(me|this\s+agent|this\s+service|my\s+endpoint)`.
